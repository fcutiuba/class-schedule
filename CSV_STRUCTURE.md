# CSV Input Structure Guide

To ensure the Presentation Assignment System runs correctly, your input files in the `files/` directory must follow the structures outlined below. The system is designed to be robust, but it relies on specific column headers to identify data.

---

## 1. All students.csv
This is the master roster of all students in the school.

| Required Column | Description | Examples |
| :--- | :--- | :--- |
| **Email** | Unique identifier (normalized to lowercase). | `jsmith@school.org` |
| **Student name** | Display name for rosters and schedules. | `Smith, John` |
| **Grade level** | Used for tie-breakers (Underclassmen prioritized). | `9`, `10`, `11`, `12` or `Freshman` |

**Example:**
```csv
Email,Student name,Grade level,Advisor
jsmith@school.org,"Smith, John",9,Mr. Henderson
ajones@school.org,"Jones, Alex",11,Ms. Miller
```

---

## 2. Presenter names.csv
Used to exclude students who are presenting from being assigned as audience members.

| Required Column | Description |
| :--- | :--- |
| **E-mail 1 - Value** | The email address of the presenter. |

**Example:**
```csv
Name,E-mail 1 - Value
"Doe, Jane",jdoe@school.org
```

---

## 3. Audience Sign Up.csv
Contains student preferences. Each category (Academic, Culture, etc.) must have exactly 5 choice columns.

| Required Column | Description / Pattern |
| :--- | :--- |
| **Email Address** | Must match the email in the master roster. |
| **Choice Columns** | Headers must match the pattern: `Which presentation is your [first/second/third/fourth/fifth] choice for the [Category] block?` |

**Example Headers for "Academic" Block:**
- `Which presentation is your first choice for the Academic block?`
- `Which presentation is your second choice for the Academic block?`
- ... (up to fifth)

---

## 4. Presentations.csv
The list of all available presentations.

| Required Column | Description |
| :--- | :--- |
| **Presentation Category (choose the best fit)** | The block name (e.g., `Academic`, `Culture`). The system singularizes and capitalizes these (e.g., "Academics" becomes "Academic"). |
| **Name of your presentation...** | Exact header: `Name of your presentation: "Creative Title: Descriptive Title"  (15 words max)` |
| **Room #** | The room where the presentation is held. |
| **Teacher Advisor Column** | Exact header: `All presentations will be paired with a teacher advisor. Is there a teacher already associated with your presentation, such as club advisor or class teacher? If yes, please write their last name below. ` |

**Example:**
```csv
Presentation Category (choose the best fit),Room #,"All presentations will be paired...","Name of your presentation..."
"1 Academics",101,Henderson,Modern Physics: A Deep Dive
"2 Culture",205,Miller,History of Salsa Dance
```

---

## Critical Rules

1. **Exact Headers**: For `Presentations.csv` and `Audience Sign Up.csv`, the headers are often generated by Google Forms. If you change the form question, you **must** update the corresponding regex in `scheduler/preprocess.py` or ensure the header matches the expected string exactly.
2. **Email Consistency**: Ensure emails are consistent across all four files. The system handles whitespace and case, but not typos (e.g., `j.smith` vs `jsmith`).
3. **Category Names**: When running the CLI with `--category`, the name must match the words found in the `Presentations.csv` category column (e.g., if the CSV says "Arts & Crafts", use `--category Arts`).
4. **No Empty Choice Columns**: Every category you wish to process must have all 5 choice columns present in the signup CSV.
5. **Independent Block Assignments:** The scheduler is unable to mandate back-to-back classes. Each block is scheduled independently, so you cannot require a "Part 1" and "Part 2" sequence.
